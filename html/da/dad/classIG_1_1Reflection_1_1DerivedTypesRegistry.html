Detailed Description

Registry of type data for types derived from BaseType .

For each type in the registry, an InfoType object is registered that contains data and eventually some utilities related to that type, including the corresponding Type object.

Type can be registered under one or more type names, representing access key to the correspongding type info.

The registry of types provides services such as registration of type data under the specified type name(s), conversion of registered type names to the appropriate Type objects (such that they can be used in reflection, e.g. for creation of object instances), creation of object instances for types with parameterless constructor, etc.

Typically, a registry will be implemented as static property in some base class.

Template Parameters
BaseTypeBase type of the types that can be registered in the current registry.
InfoTypeType of objects that hold information on the registered type and eventually some utilities for that type. Must derive from TypeInfoBase.

Registers the type information typeData under the name typeName .

Type (TypeInfoBase.ClassType) must be derived from (or equal to) BaseType . Each specific type can be registered with several type names.

It is preeferable to use Register(InfoType) for type registration because this method registers the type under its actual names (simple, fully qualified and assembly qualified).

Parameters
