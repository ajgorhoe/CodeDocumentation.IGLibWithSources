201  // The current algorithm causes clustering because when indices are already clustered at some region (which may occur

203  // corrected algorithm should somehow renumber all unoccupies indices and generate each new index for a unique consecutive
204  // position among unoccupied indices counted in this way. In order to finc correct gaps for newly created indices, a kind
205  // of modified bisection can be used (because for each index currently in the list, it can be calculated which would be
